# Hygiene Rules (写作与卫生约定)

> **触发场景**：任何 Markdown 写入操作。

## 1. 命名规范
- **文件名强制中文**：所有 `wiki/` 下的文件名必须保存为中文。
- **资产命名全英文**：`assets/` 下的图片名必须全英文、小写、无空格（使用 `-` 分隔）。

## 2. Wikilink (路径校准)
- **禁止绝对前缀**：严禁使用 `[[wiki/xxx]]`。
- **正确格式**：使用相对于 `wiki/` 根目录的路径，即 `[[目录/文件名.md]]`（例如 `[[games/原神.md]]`）。

## 3. 质量标准
- **Stub 清理**：禁止创建仅含标题的空白笔记。内容不足 20 字的占位符应直接合并或删除。
- **引用确权**：每个重要论断必须注明来源 `[来源]`。
- **动图引用**：使用 `![描述](assets/xxx.gif)` 格式。

## 4. 评估维度 (UX)
涉及交互手感评估时，必须覆盖：**响应感、打击感、节奏感、容错感、重量感**。
