## 1. 系统概览与页面矩阵

| 页面/模块 | 功能定位 | 核心交互 | 权重 |
| :--- | :--- | :--- | :--- |
| **欢迎弹窗** | 情感激活 + 即时赠礼 | 单一CTA引导 | P0 |
| **回归进程** | 进度可视化核心 | 里程碑树 + 任务领取 | P0 |
| **归途助力** | 难度追赶机制 | 专属BUFF展示 | P1 |
| **七日签到** | 每日福利 | 线性7格签到 | P0 |
| **归路指引** | 内容导航防迷失 | 快速跳转各玩法 | P1 |

## 2. 视觉层级与布局范式
- **双栏信息架构**：进程页将"进度树（左）"与"任务清单（右）"并排，玩家一屏内同时看到"我在哪"和"该做什么"。
- **树状进程可视化**：分叉里程碑树替代线性进度条，每节点标注所需点数和奖励，令长线目标有清晰分解感。
- **欢迎弹窗勾选反馈**：即时赠礼覆盖蓝色勾选图标，解决"关弹窗后物品去哪了"的焦虑。

## 3. 交互闭环与链路推导
- **单一入口强引导**：欢迎弹窗只有一个CTA，不提供自由探索选项，体现"效率最大化"导向。
- **完整链路**：弹窗领礼 → 进程主页识别缺口 → 任务清单执行 → 归路指引上手新玩法 → 归途助力降低门槛。

## 4. UX 诊断
- **亮点**：过程透明度极高，玩家始终清楚"完成了多少/还差多少"，每个任务一键跳转对应系统。
- **痛点**：左侧有6个子模块，信息量对刚回归玩家略重，可能产生决策疲劳。
