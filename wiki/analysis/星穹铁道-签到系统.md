## 1. 系统概览与页面结构 (System Overview)
- **场景属性**：属于“旅情事记”活动合集下的一个二级子页面。设计风格极其明亮、通透。
- **功能定位**：版本更新后的核心福利发放口，通过高额的“星轨专票”投放确保版本首周的高留存率。

| 页面/组件 | 功能定位 | 视觉权重 |
| :--- | :--- | :--- |
| 三月七立绘 | 角色亲和力引导，与“邀游”主题呼应 | P0 |
| 非对称奖励矩阵 | 核心领奖区，通过 2x3 + 1 的排布拉大第 7 天的权重 | P0 |
| 左侧活动列表 | 列表式导航，展示其他并行活动及红点提醒 | P1 |

## 2. 视觉层级与布局范式 (Hierarchy & Layout)
- **视觉母板 (Thematic)**：白色作为底色，配以淡蓝色的微缩背景和彩色的 UI 装饰元素（如三月七的相机闪光动效）。
- **非对称格位布局**：
    - 前 6 天采用 2x3 的矩形格位，占据中央区域。
    - **第 7 天** 采用一个高度翻倍的长方形格位，视觉宽度虽窄但垂直高度最高，且奖品数量显著（x3），形成了极强的视觉锚点。
- **自适应逻辑**：典型的 16:9 扁平化布局，通过左侧导航栏腾出中央大面积的展示空间。

## 3. 交互闭环与链路推导 (Interaction & Flow)
- **单页闭环**：玩家在活动合集中选中“邀游天外” -> 自动展开右侧面板 -> 点击“领取”。
- **引导动效**：Day 1 按钮带有“向下箭头”和“领取”文案，通过微弱的位移动效引导用户。
- **跳转逻辑**：由于是活动合集的一部分，用户可以无缝在“签到”与“角色试用（锋芒崭露）”间切换。

## 4. 系统级 UX 诊断 (UX Diagnosis)
- **设计亮点**：**极简主义与信息清晰度**。没有任何冗余的修饰，玩家一眼就能识别出“我能领什么”和“我什么时候能领”。
- **潜在问题**：活动剩余时间（40天）在右上角字号极小，用户可能无法直观感受到活动的限时紧迫感。
- **对比回归系统**：相较于回归系统的“引导感”，常规签到更强调“福利感”。
