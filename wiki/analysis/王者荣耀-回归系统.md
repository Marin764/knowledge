## 1. 系统概览与页面矩阵

| 页面/模块 | 功能定位 | 核心交互 | 权重 |
| :--- | :--- | :--- | :--- |
| **回归特权** | 核心钩子 / 皮肤体验 | 皮肤选择 + 对战激活延伸 | P0 |
| **签到福利** | 每日留存 | 7 格签到 + 终极大奖 | P0 |
| **送皮肤** | 即时激活 / 情感礼物 | 一键领取免费皮肤 | P0 |
| **英雄情报** | 内容导航 / 版本引导 | 推荐英雄快速跳转 | P1 |
| **好友近况** | 社交钩子 | 查看好友战绩触发 FOMO | P1 |

## 2. 视觉层级与布局范式
- **皮肤试用核心策略**："回归特权"中5款史诗皮肤的并排展示，让玩家直接"试穿"自己心仪的皮肤。这是 MOBA 类回归系统独有的高效钩子——玩家为了继续使用喜欢的皮肤，必须保持活跃。
- **"每打 1 局+1 天"行为绑定**：将皮肤体验时长与对战行为直接绑定，是极其巧妙的留存机制。用户不再只是"领奖励"，而是"为了皮肤去打比赛"，这将回归系统与核心玩法深度融合。
- **社交钩子**：顶部"好友近况"页签是主动 FOMO 工具，让玩家看到好友在玩什么，激发竞争欲和社交归属感。

## 3. 交互闭环与链路推导
- **回归钩子链路**：
  1. 收到推送通知"回来领取史诗皮肤" → 进入回归福利主页
  2. 看到5款史诗皮肤免费试用 → 选择最心仪的皮肤
  3. 开始对战，每局自动延长体验时间（留存循环）
  4. 第7天签到领取终极大奖（排名奖励）
  5. "英雄情报"引导了解版本 Meta，降低复出焦虑

## 4. UX 诊断
- **核心亮点**：**皮肤试用机制是 MOBA 回归设计的标杆**。将高价值内容（史诗皮肤）免费开放体验，显著降低了回归门槛，同时通过行为绑定延长了效果周期。
- **亮点**：送皮肤模块根据玩家历史偏好推荐，体现了个性化运营能力。
- **痛点**：签到福利的第 1-6 天奖励（排位保护卡、加星卡）对于已久未玩的玩家来说意义有限，可以考虑将前期奖励替换为更通用的资源。
