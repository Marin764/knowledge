# 每日任务系统

## 概述

每日任务（Daily Missions）是一种以**每日为周期**的留存与参与机制，通过要求玩家完成特定游戏目标（而非仅仅登录）来驱动深度参与。相比每日登录奖励，它能将"表面的打开行为"转化为"实质性的游戏时长消耗"，从而提升长期留存质量。

## 每日登录奖励 vs. 每日任务

| 维度 | 每日登录奖励 | 每日任务 |
|------|------------|---------|
| 玩家参与度 | 被动（只需打开App） | 主动（必须完成游戏目标） |
| 对留存的贡献 | 基础（促使打开App） | 深度（促使实质性游玩） |
| 推送通知质量 | 低（空洞提醒） | 高（基于具体进展） |
| 新手引导价值 | 无 | 有（可设计强制道具使用实验） |
| 奖励感知价值 | 低（得来全不费功夫） | 高（须付出努力才获得） |

## 核心价值

1. **约定机制（Appointment Mechanic）**：时间限制 + 特定奖励 = 习惯性回访
2. **漏斗缩短**：将任务显性放置于核心 UI，相比埋在活动中减少触达路径
3. **推送质量提升**：基于任务进展的推送比"来领礼物"更有行动驱动力
4. **新手引导优化**：强制使用道具的任务帮助玩家建立信心，理解游戏机制

## 设计要点与实现建议

### 1. 目标设计
任务应与核心玩法紧密绑定，避免与核心循环偏离。
- **《原神》每日委托**：消耗体力/推进角色培养（与核心绑定）。
- **《使命召唤手游》**：武器升级/游戏内货币（与核心绑定）。

### 2. 结构分类
- **序贯式任务链**：各任务按顺序完成，后者解锁于前者完成后。如《Candy Crush Saga》的 Classic Chocolate Box 事件，四关卡必须依次通关才能到达终点，创造"接近完成"的压力感。
- **多路径可选任务**：同一时间提供多个任务选项，玩家可自由切换。如《Toy Blast》Hoop Shot 事件提供四个并行选项，随时可切换策略，降低"卡关导致放弃"的风险。
- **团队协作任务**：任务进度归属团队，贡献可见，引入同伴压力。如《Project Makeover》的每周团队任务，贡献榜触发竞争动力。
- **赛季通行证集成**：任务与赛季进度系统融合。如《Toon Blast》的 mini-season pass，通过短期通行证结构提供"低投入-高成就"的任务体验。

### 3. 展示位置
**显性展示优于隐性活动**。在移动端遵循“少点击 = 大影响”原则，将每日任务置于主屏，尽量减少触达漏斗的点击层级。

### 4. 协同机制
- **与每日商店特惠结合**：如《Time Blast》，将任务与每日商店优惠绑定，引导玩家每日访问商店。
- **与推送通知联动**：基于具体进展的通知（“还差一局即可领取奖励”）远比空泛的登录提醒有效。
- **新手引导价值**：可设计强制使用道具的任务，建立玩家对 Power-ups 的信心。
