# 回归系统 (Recall / Return System)

回归系统是游戏运营中用于挽回流失玩家（Churned Players）的核心机制。通过情感补偿、进度追赶和经济激励，降低回流玩家的“断代感”与“挫败感”，从而实现二次留存。

## 1. 核心模块拆解

### 1.1 身份与仪式感 (Identity)
- **回流触发条件**：通常定义为“连续离线天数 > 14/30/90天”。
- **欢迎界面**：专属的 Loading 图或开场剧情（如《原神》的星之归还剧情），重塑情感链接。
- **视觉标签**：为回流玩家提供限时的专属头像框或昵称标识，增强社区归属感。

### 1.2 签到与存留 (Sign-in)
- **分阶段奖励**：7/14 日签到。
- **高价值前置**：核心奖励（如抽卡券、稀有皮肤）通常放在第 1-3 天，用于在第一时间内锁住回流意愿。

### 1.3 进度追赶 (Catch-up)
- **数值 Buff**：回归期间副本掉落翻倍（Double Drop）、角色经验加成。
- **资源包**：一次性发放大量体力、养成基石材料，对冲由于离线导致的版本战力脱节。

### 1.4 引导与复训 (Tutorial)
- **新版本综述**：通过极简图文汇总离线期间的新系统、新地图。
- **回归任务链**：将回归奖励与核心玩法绑定，引导玩家快速进入当前的游玩循环。

## 2. UX 设计准则

### 2.1 路径极简化
- **原则**：回流玩家的耐心极低。
- **实现**：回归入口必须在主界面（HUD）高亮显示，且任务操作链路不得超过 3 次跳转。

### 2.2 先给后取 (GIVE before GET)
- **原则**：用即时正向反馈冲抵挫败感。
- **实现**：登录即领第一波大奖，后续奖励再由任务触发。

### 2.3 难度缓冲区
- **原则**：手感生疏是流失的隐形推手。
- **实现**：在竞技类游戏中，短期内会降低匹配分（MMR），提供一个“虐菜”或“简单模式”的过渡期。

## 3. 典型案例

| 游戏 | 系统名称 | 核心亮点 |
| :--- | :--- | :--- |
| **《原神》** | 星之归还 | 强化的剧情仪式感 + 武器/角色四选一（早期版本）。 |
| **《星穹铁道》** | 巡星之礼 | 与模拟宇宙深度绑定的双倍掉落。 |
| **《王者荣耀》** | 英雄回归 | 极其精细的匹配难度下调机制 + 全英雄限免试玩。 |
