# ADR 008: 批量资产摄入转向全量识别先行

## 背景
在处理大规模游戏 UI 素材（如 >100 张截图）时，原有的“抽样审计”机制虽能快速锁定 UID，但容易遗漏离散分布的核心系统原型。

## 决策
1.  **废弃抽样审计**：不再以 3-5 张图的抽样作为开端。
2.  **强制全量识别**：AI 在处理新素材包时，必须先对 `raw/` 目录下所有原始 ID 进行语义分析。
3.  **日志先行**：全量对照表必须在物理提取前在 `mapping_log.md` 中构建完成。

## 原因 (Why)
- **全局最优去重**：只有看到全集，AI 才能精准判断哪一张是“原型”，哪几张是“过程图”。
- **系统覆盖度保证**：确保不会因为抽样偏差而漏掉“行迹树”、“结算页”等关键但数量较少的界面。

## 影响
- AI 在摄入初期的 Token 消耗和处理时间会增加。
- 资产库的“信噪比”将显著提升，确保 `assets/` 目录只有高质量的原型图。

## 状态
Accepted (2026-04-24)
