# ADR 024: 三国杀 UI 资源拆分为组件与组合模式参考包

## Status

accepted

## Date

2026-04-30

## Context

`assets/three_kingdoms_kill_controls/official_components/` 已经包含一批官方组件图和整页模板图，但它们混放在同一目录下。对于 AI 生图任务来说，这会带来两个问题：

- agent 难以区分“单控件参考”和“整页组合模式参考”。
- JSON 说明虽然已有初步语义，但缺少面向生成任务的 `category / layout_role / do_use / do_not_use / pairs_with` 等字段。

用户的目标不是把这些图做成可直接使用的透明素材，而是让 agent 能理解并参考这些 UI 控件来生成新的三国杀风格界面。

## Decision

在 `assets/three_kingdoms_kill_controls/` 下新增两层主读取结构：

- `components/`：存放按钮、Tab、弹窗、图标、状态角标等单控件或同类状态合集。
- `patterns/`：存放战斗 HUD、抽卡舞台、通用系统页、活动页、武将皮肤页等组合模式图。

同时新增面向 AI 生图的语义清单：

- `metadata/components/sgs-components-for-ai.json`
- `metadata/patterns/sgs-patterns-for-ai.json`

保留 `official_components/` 作为来源追溯目录，不作为 agent 默认第一读取入口。

## Consequences

- 新 agent 可以先读说明与 JSON，再查看对应小图，减少把整页模板误当单控件的概率。
- 三国杀资源从“图片包”升级为“组件语义 + 组合模式 + 整页截图补充”的参考包。
- 当前仍未进入完整截图识别链，暂不建立 `crops / review / atlas` 层；后续如需要控件级精度，可继续从 `three_kingdoms_kill_ui/` 补 bbox 裁切与复核图。
