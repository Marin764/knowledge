# ADR 006: 回归系统 HTML 原型改为 16:9 游戏界面稿

## 背景
此前生成的 `exports/回归系统-网页原型.html` 将回归系统转译为网页原生展示页，重点强调固定导航、纵向清单和展开面板。这一方向适合“专题页/官网页”场景，但不符合当前任务目标。

当前目标不是做网页专题，而是**根据回归系统规范生成一个游戏界面**，只不过使用 HTML/CSS 进行表现。因此，原先“网页原生交互优先”的决策需要在这个具体原型任务上被修正。

## 决策
1. **删除网页专题式原型**：移除 `exports/回归系统-网页原型.html`。
2. **改为 16:9 游戏界面稿**：新增 `exports/回归系统-界面原型_V1.html`，以 16:9 舞台为基础，采用游戏 UI 的分区思路组织界面。
3. **保留规范结构，切换表现形式**：继续遵循回归系统规范中的页面合同、组件合同和状态逻辑，但在表现上回到游戏界面的 HUD、模块导航、主舞台和奖励侧栏。
4. **将 ADR 005 视为被本条 supersede 的特定实现**：ADR 005 仍保留记录价值，但不再作为当前回归系统 HTML 原型的实现方向。

## 原因 (Why)
- **贴合当前意图**：用户要的是“游戏回归系统界面”，不是官网活动页。
- **验证规范的另一种下游能力**：证明同一份规范不仅能生成网页专题，也能生成更接近游戏成品界面的 HTML 稿。
- **便于后续细化**：16:9 HTML 稿更适合作为 UI 设计草模、交互排布基底和后续 Figma/前端重构参考。

## 影响
- `exports/` 中回归系统的 HTML 资产从“网页原型”切换为“游戏界面原型”。
- 后续如果继续生成战令、签到等 HTML 稿，需要先明确目标是“网页承载”还是“游戏界面表现”。
