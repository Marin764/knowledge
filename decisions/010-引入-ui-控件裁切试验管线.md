# ADR 010：引入 UI 控件裁切试验管线

- Status: accepted
- Date: 2026-04-24

## Context

当前知识库已经具备：

1. `assets/<game>_ui/` 级别的原型截图资产库。
2. `ui_analysis -> ux_spec -> prototype` 的系统级结构化分析链路。

但仍缺少一条专门面向“截图资源拆分”的能力：

- 无法把单张截图中的按钮、Tab、货币条、关闭按钮、主面板等控件拆成独立资源。
- 无法为后续 Figma 重建、设计资源沉淀或训练集整理提供 bbox 与裁切图。
- 现有 `skill 01` 和 `skill 02` 主要回答“系统怎么组织”，而不是“这一屏有哪些可复用控件”。

## Decision

引入一条最小可行的 UI 控件裁切试验管线：

1. 新增目录 `assets/star_rail_controls/` 作为星铁控件裁切试验区。
2. 使用手动 `bbox` 标注作为第一阶段输入，而不是直接依赖自动检测。
3. 新增脚本 `scripts/images/extract_ui_crops.ps1`，用于：
   - 读取 manifest
   - 批量导出裁切单图
   - 生成带框复核图
   - 生成 contact sheet 总览图
4. 第一批示范资产采用：
   - `assets/star_rail_controls/manifest-shop-main.json`
   - 来源图：`assets/star_rail_ui/hsr-shop-main.jpg`

## Consequences

### Positive

- 知识库首次具备“截图 -> 控件裁切图”的物理产出能力。
- 可以直观看到 bbox 粒度是否合理，再决定是否继续扩展到透明背景或更多截图。
- 后续更容易衔接 Figma 组件重建和设计资源归档。

### Tradeoffs

- 这仍是手动标注驱动的 MVP，不是自动识别。
- 当前导出的资源以矩形裁切为主，并未处理透明背景。
- contact sheet 更适合复核与分类，不等于最终设计组件库。

## Follow-up

- 若示范效果可接受，下一步扩展到更多星铁截图。
- 在控件 manifest 中补充更稳定的命名、状态和用途字段。
- 后续再评估是否增加透明背景导出和 `ui_resource_map` 类型文档。
