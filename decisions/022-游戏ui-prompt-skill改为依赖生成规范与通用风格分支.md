# ADR 022：游戏 UI Prompt Skill 改为依赖生成规范与通用风格分支

- Status: accepted
- Date: 2026-04-28

## Context

上一版 `skills/06-game_ui_prompt_template.md` 已经被收束为精简执行卡，但仍有三个问题：

1. 描述上容易让人理解成“与 Skill 02 配合使用”，而不是“与 Skill 02 产出的设计规范配合使用”。
2. `16:9` 被表述得过于接近“固定横屏”，没有明确“横屏优先，但可按系统需要做竖屏变体”。
3. 风格选项仍带有较强的回归系统示例痕迹，不够通用。

## Decision

对 `skills/06-game_ui_prompt_template.md` 做进一步调整：

1. 明确依赖对象是**生成后的设计规范文档**，优先读取 `exports/*交互设计规范*.md`。
2. 将比例约束改为**默认 16:9，横屏优先，但不强制锁死方向**。
3. 删除不必要的“默认把哪些材料一起视为输入”的描述。
4. 将风格分支改为通用提炼规则，由 agent 从设计规范和分析页中动态归纳候选风格，而不是写死为某个系统的固定选项表。

## Consequences

正面影响：

- Skill 与“设计规范产物”之间的关系更清晰。
- 适配横屏与竖屏系统时更自然。
- 风格提问规则更通用，适合后续扩展到其他系统。

成本：

- agent 在多风格系统里需要先做一次较轻量的风格提炼，再问用户偏好。

## Follow-up

- 后续可考虑把“常用风格分支字典”独立沉淀成附录型参考，而不是写死在主 Skill 中。
