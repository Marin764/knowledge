# ADR 017: 引入竖屏 (Portrait) 空间构图模型

## 日期
2026-04-27

## 状态
已接受 (Accepted)

## 上下文 (Context)
在之前的 UI 分析管线（ADR 009）中，我们建立了基于 16:9 横屏布局的 `Spatial Slot` 映射体系。
随着“剑与远征：启程”等竖屏游戏的接入，现有的槽位定义（如 `left_rail` / `right_panel`）在纵向堆叠结构下显得不直观且难以准确指导生成。

## 决策 (Decision)
针对竖屏 UI，我们引入一套专门的 **Vertical Spatial Model (VSM)**。该模型不再强调左右布局，而是强调由顶至底的纵向功能分层：

1.  **top_bar (顶栏)**：系统信息监控（货币、等级）、全局导航（标题、返回）。
2.  **hero_stage (上部舞台)**：高画幅视觉重心。通常放置角色立绘、3D 模型展示或沉浸式场景。
3.  **content_panel (内容面板)**：交互核心区。承载网格列表、卡片组、详情文本等密集信息。
4.  **bottom_bar (底部导航)**：拇指舒适区的系统级/页面级切换 Tab。

## 后果 (Consequences)
**正面：**
- **布局一致性**：为竖屏界面的分析提供了统一的术语表。
- **生成精度**：在下游 AI 生成环节（如 Midjourney / GPT-4V），可以明确使用 `top-to-bottom stack` 而不是 `left-to-right flow` 进行约束。

**负面：**
- 增加了一套术语，需要在 `skills/01-ui_analysis.md` 中补充对应的映射逻辑，以防分析者混淆横竖屏模型。
