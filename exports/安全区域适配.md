# "安全区域适配"

"在移动端游戏中，为避开设备物理遮挡（刘海、打孔、圆角）和系统交互区（Home Indicator）而进行的UI位置调整和防穿帮设计。"

## 核心机制

随着全面屏（19.5:9及以上）和异形屏的普及，游戏 UI 不能再简单地铺满全屏，必须遵守系统定义的“安全区域”（Safe Area）[来源: 【游戏交互】手游适配设计.md]。

## 常见适配问题及方案

这通常需要结合 [[wiki/concepts/动态UI适配.md|动态UI适配]] 和 [[wiki/concepts/UI锚点.md|UI锚点]] 一起解决：

### 1. 物理遮挡（刘海/圆角）
- **问题**：贴边的UI元素（如左上角的返回按钮）容易被遮挡或被圆角截断。
- **方案**：整体 UI 向内偏移。同时，为了防止向内偏移后露出黑边（穿帮），需要对背景底板进行向外延伸或补全 [来源: 【游戏交互】手游适配设计.md]。

### 2. 系统手势（Home Indicator）
- **问题**：iOS 底部的横条区域（Home Indicator）容易与游戏内底部的交互（如上滑出牌、底部菜单栏）产生手势冲突。
- **方案**：
  - **交互层面**：尽量避免在游戏底部使用“从下往上滑”或“底部横向滑动”的手势。例如《炉石传说》在早期就遭遇过滑动冲突，后续通过放大热区缓解；而《哈利波特》则直接将卡牌上移 [来源: 【游戏交互】手游适配设计.md]。
  - **视觉层面**：底部菜单需整体上移（非贴边菜单），或者将 UI 底板向下补全以防止穿帮（贴边菜单）[来源: 【游戏交互】手游适配设计.md]。

### 3. 场景化 UI 穿帮
- **问题**：当 UI 是一本书、一个卷轴等带有边缘的场景化物体时，从 16:9 拉伸至全面屏，如果只用锚点居中，会导致两边突然被截断。
- **方案**：在美术设计时就要考虑到全面屏的尺寸极限（如设计尺寸设定为 1624*1000 以涵盖最大可能），或者在底部叠加一层纯色遮罩用于补色 [来源: 【游戏交互】手游适配设计.md]。
